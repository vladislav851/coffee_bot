# coffee_bot

Telegram-бот для учёта товарных остатков между складом и кофейней: перемещение позиций, запрос остатков, инвентаризация и справочник КБЖУ. Построен на `aiogram` 3 (async, FSM) и `aiosqlite`.

## Возможности

- **Перемещение остатков** — свободный текст вида `молоко 4 л` переносит количество между складом и кофейней с учётом направления, блокирует уход в минус.
- **Запрос остатков** — по названию товара (или без чисел) показывает текущий остаток; отдельные отчёты по локации и сводный отчёт по всем товарам.
- **Инвентаризация** — пакетный ввод списка товаров перезаписывает остатки на выбранной локации.
- **КБЖУ** — нечёткий поиск (`rapidfuzz`) по названию блюда возвращает калорийность и БЖУ.
- **Отмена последнего действия** (`undo`) и авто-очистка чата от сообщений бота, чтобы не засорять переписку отчётами.

## Стек

- Python 3.11+, `aiogram` 3 (Dispatcher/FSM), `aiosqlite` (WAL), `rapidfuzz` (нечёткий поиск), `openpyxl` (импорт данных из Excel).

## Структура проекта

```
handlers/     — обработчики сообщений и callback-кнопок (по сценариям)
services/     — парсинг ввода, fuzzy-поиск, доступ к БД, антизасорение чата
keyboards/    — inline-клавиатуры
states/       — FSM-состояния
database/     — схема SQLite и подключение
scripts/      — разовые скрипты импорта данных из Excel
deploy/       — systemd unit и инструкция по разворачиванию на сервере
```

База данных `database/inventory.db` создаётся автоматически при первом запуске (см. `database/schema.sql`), изначально пустая — товары добавляются через сценарий инвентаризации в самом боте либо скриптами импорта в `scripts/`.
